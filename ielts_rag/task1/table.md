# Table — Type-Specific Evaluation Guide

## Visual identification

Rows and columns of **numerical or categorical data**. May show single period or changes across columns (years).

## Task Achievement priorities

| Priority | What to assess |
|----------|----------------|
| **High** | Patterns across rows/columns; highest/lowest; notable contrasts |
| **Medium** | Logical grouping (by row category or time) |
| **Low** | Reading table row-by-row mechanically |

## Overview expectations

Summarise the **main pattern** visible in the table (e.g. "Overall, all countries saw growth in exports except Country C."). No exhaustive data in overview.

## Body paragraph grouping

- Group by **category** (compare countries/products)
- Group by **time period** when columns show years
- Highlight **exceptions** to the general pattern

## Language focus

stood at, recorded, reached, remained stable, increased to, declined to, ranked, accounted for, respectively, in contrast, similarly

If columns show time progression, trend language is appropriate. If static, use comparison language.

## Common weaknesses

- Sequential row reading without synthesis
- Missing the clearest pattern
- Overloading with every cell value
- No overview of cross-table trend

## Data accuracy checks

Cross-reference row/column intersections. Transposed figures = **Data accuracy** mistakes.

## Model report tone

Pattern-recognition focused, selective with figures. Target **160–190 words**.
