# Multiple Diagrams — Type-Specific Evaluation Guide

## Visual identification

**Two or more similar diagrams** — typically two maps (different years), before/after layouts, or multiple related process stages shown separately.

## Task Achievement priorities

| Priority | What to assess |
|----------|----------------|
| **High** | Comparison between diagrams; major differences/developments; strong overview |
| **Medium** | Grouping changes logically; unchanged elements noted |
| **Low** | Describing each diagram separately without comparison |

## Overview expectations

State the **main change or difference** across diagrams (e.g. "Overall, the town centre was significantly redeveloped between 1990 and 2020, with new housing and transport infrastructure."). For two maps: transformation summary.

## Body paragraph grouping

- **By area** comparing both time points
- **By type of change** across both diagrams
- Avoid: "In the first diagram… In the second diagram…" without synthesis

## Language focus

For maps: passive change vocabulary (was built, was removed, was expanded).
For dual processes: comparative sequencing (whereas, in contrast, similarly, while the first diagram shows…, the second illustrates…).

## Common weaknesses

- Sequential description without comparison language
- Missing major developments visible only by comparing both
- Weak or single-diagram overview
- Treating as unrelated descriptions

## Data accuracy checks

Changes must be verified against **both** diagrams. Features present in only one period must be described correctly.

## Model report tone

Comparative, developmental. Target **170–200 words**.
